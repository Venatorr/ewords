  1. http://127.0.0.1:8000/api/signup/

    • через GET отображаются все текущие пользователи
    • после регистрации вернется токен, можно его использовать
  2. http://127.0.0.1:8000/api/obtain-jwt-token/
    • метод GET не предусмотрен
    • через POST можно по имени пользователя и паролю получить токен
  3. http://127.0.0.1:8000/api/refresh-jwt-token/
    • метод GET не предусмотрен
    • через POST можно по текущему токену получить новый, с новой датой экспирации (не тестил так ли это)
  4. http://127.0.0.1:8000/api/verify-jwt-token/
    • метод GET не предусмотрен
    • через POST можно проверить токен (хз зачем может быть нужно)
  5. http://127.0.0.1:8000/api/languages/
    • метод GET возвращает все языки
    • через POST можно добавить свой язык (вводится только поле name)
    • Доступен любому пользователю
  6. http://127.0.0.1:8000/api/words/
    • метод GET возвращает все слова, если ты неавторизован через токен,
      либо слова текущего пользователя, авторизованного через токен
    • через POST можно добавить слова, но только авторизованному пользователю
      хз как тесить через браузер, через postman надо выбрать authorization -> bearer token и заполнить токен
      после этого post будет выполняться
  7. Общая структура работы с API на примере Words:
    • Для перехода по страницам добавляем ?page=
      например, http://127.0.0.1:8000/api/words/?page=2
    • Для update существующей строки или удаления открываем добавляем id нужной строки
      http://127.0.0.1:8000/api/words/7/ - слово с id=7
    • Для массового добавления используется POST и JSON пишется так: [{...}, {...}, {...}]
